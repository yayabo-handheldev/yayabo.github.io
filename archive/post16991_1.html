相比用Cygwin建立PSP开发环境, 我觉得用devkitPro较为容易.

首先下载devkitProUpdater, 到这里下载:
[url=http://www.devkitpro.org/]devkitPro[/url]

执行devkitProUpdater后, 设定了基本的项目, 就会来到开发环境的选择:
[attach]55[/attach]
除了PSP, devkitPro 还有GBA, NDS等的开发环境.

选择好, devkitProUpdater就会到网上下载开发环境的配件.

安装完差不多就可以用了, 但devkitProUpdater忘了把"devkitPSP/bin"放到PATH里, 所以我们可以自己把它放进去, 或是建立一个像下面一样的START.BAT, 放在devkitPro的主目录内, 设定 "devkitPSP/bin" 并同时打开COMMAND PROMPT:
set path=%path%;C:/devkitPro/devkitPSP/bin;
cmd
(11月4日註: 終於明白devkitProUpdater沒有把devkitPSP/bin放在path里的原因, 要是你的系統里同時安裝了cywin+toolchain的psp開發環境, 把devkitPSP/bin放在path里會導致它"不良於行")

我们可以随便选一个例子试一下:
[attach]56[/attach]
注意:要是你之前安装过PSP开发环境, 要把旧的环境参数"PSPDEV"或"PSPSDK"清除掉.

再附上我自己正在用的START.BAT. 要是devkitProUpdater没把环境参数设定好,可以用这个. 另外, 安装时一定要装"MSYS"!
set PATH=%PATH%;c:devkitPromsysin;c:devkitProdevkitPSPin
set PSPDEV=
set PSPSDK=
set DEVKITPRO=/c/devkitPro
set DEVKITPSP=/c/devkitPro/devkitPSP
CMD
